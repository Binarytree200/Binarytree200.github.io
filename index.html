Linux常用操作命令 (vi/vim编辑器篇) 1.5

发表于2017-12-05更新于2019-10-01

vi有3个模式：插入模式、命令模式、底行模式。
插入模式：在此模式下可以输入字符，按ESC将回到命令模式。
命令模式：可以移动光标、删除字符等。
底行模式：可以保存文件、退出vi、设置vi、查找等功能(低行模式也可以看作是命令模式里的)。
注意;在使用底行模式（又称命令模式它俩可以理解成一个）之前，请记住先按 ESC 键确定您已经处于命令行模式下后，再按冒号 : 即可进入底行模式。

使用方法大全

打开文件、保存、关闭文件(vi命令模式下使用)

vi filename //打开filename文件
:w //保存文件
:w fwblog.xin //保存至fwblog.xin文件
:q //退出编辑器，如果文件已修改请使用下面的命令
:q! //退出编辑器，且不保存
:wq //退出编辑器，且保存文件

插入文本或行(vi命令模式下使用，执行下面命令后将进入插入模式，按ESC键可退出插入模式)

a //在当前光标位置的右边添加文本
i //在当前光标位置的左边添加文本
A //在当前行的末尾位置添加文本
I //在当前行的开始处添加文本(非空字符的行首)
O //在当前行的上面新建一行
o //在当前行的下面新建一行
R //替换(覆盖)当前光标位置及后面的若干文本
J //合并光标所在行及下一行为一行(依然在命令模式)

移动光标(vi命令模式下使用)

1、使用上下左右方向键
2、命令模式下：
h 向左、j 向下、k 向上、l 向右。
空格键向右、Backspace向左、Enter移动到下一行首、-移动到上一行首。

删除、恢复字符或行(vi命令模式下使用)

x //删除当前字符
nx //删除从光标开始的n个字符
dd //删除当前行
ndd //向下删除当前行在内的n行
u //撤销上一步操作
U //撤销对当前行的所有操作

搜索(vi命令模式下使用)

/cszhi //向光标下搜索cszhi字符串
?cszhi //向光标上搜索cszhi字符串
n //向下搜索前一个搜素动作
N //向上搜索前一个搜索动作

跳至指定行(vi命令模式下使用)

n+//向下跳n行
n-//向上跳n行
nG //跳到行号为n的行
G //跳至文件的底部

设置行号(vi命令模式下使用)

:set nu //显示行号
:set nonu //取消显示行号

复制、粘贴(vi命令模式下使用)

yy //将当前行复制到缓存区，也可以用 “ayy 复制，”a 为缓冲区，a也可以替换为a到z 的任意字母，可以完成多个复制任务
nyy //将当前行向下n行复制到缓冲区，也可以用 “anyy 复制，”a 为缓冲区，a也可以替 换为a到z的任意字母，可以完成多个复制任务
yw //复制从光标开始到词尾的字符
nyw //复制从光标开始的n个单词
y^ //复制从光标到行首的内容
y$ //复制从光标到行尾的内容
p //粘贴剪切板里的内容在光标后，如果使用了前面的自定义缓冲区，建议使用”ap 进 行粘贴
P //粘贴剪切板里的内容在光标前，如果使用了前面的自定义缓冲区，建议使用”aP 进 行粘贴

替换(vi命令模式下使用)

:s/old/new //用new替换行中首次出现的old
:s/old/new/g //用new替换行中所有的old
:n,m s/old/new/g //用new替换从n到m行里所有的old
:%s/old/new/g //用new替换当前文件里所有的old

编辑其他文件

:e otherfilename //编辑文件名为otherfilename的文件

vmware workstation（vm虚拟机）的使用

发表于2017-11-26更新于2019-10-01

vmware workstation 下载地址：http://rj.baidu.com/soft/detail/13808.html?ald

vm虚拟机简单来说就是在真实机上模拟多个计算机系统，操作与真机上的操作无异，但是它因为是在一台真机上模拟多个系统，所以它会占用大量的内存和cup，如果电脑的性能不是太强不建议用vm虚拟机
下面我来介绍vm虚拟机的使用方法

创建虚拟机有两种方式一是直接在主页上点击创建新的虚拟机二是工具栏点击文件点击新建虚拟机，
如果你已经创建完虚拟机可以在主页上点击打开虚拟机同样在工具栏文件点击打开效果是一样的

工具栏的第二个编辑 里面的剪切复制粘贴都是从真机向虚拟机内进行剪切复制粘贴
虚拟机网络编辑器 就是为了让虚拟机可以想真机一样联网而模拟出来的虚拟网卡，在虚拟网络编辑器中
有多种网络连接模式自己可以经行自定义
而首选项故名思意，就是设备的第一个选择对象，里面可以对硬件的优先级进行更改

工具栏第三个查看，为了调整用户在使用虚拟系统时的视觉感而出现的工具
这里我主要说一下unity 和 控制台视图
Unity 是VM虚拟机一种无缝显示方式 虚拟机系统 开启Unity模式后,就可以在真实机桌面上显示虚拟机系统的应用程序
控制台视图 就是在一个界面下显示多个虚拟机界面

工具栏第四个虚拟机 里面包含了电源 可移动设备 暂停 发送 快照 管理 设置
电源内的挂起计算机就在当前位置产生一个快照，等下次点继续运行客户机时，重新在快照当前保存的时刻恢复当前状态 关闭 重启按字面意思理解就可以了
在可移动设备内 可以进行可移动设备的插入弹出 比如u盘 移动硬盘等等都算是可移动设备
这个送功能就是向虚拟机内发送 ctrl+shift+del 防止与真机冲突
快照 简单点说它就相当于一个照片 下次可以从这个照片这个位置继续当前的操作
管理 里面有克隆 和 清除 克隆就是克隆出一个一模一样的虚拟机但克隆也分为两种 一个是链接克隆 就是在原来的虚拟机基础上克隆出来的虚拟机 如果原来的虚拟机被删除了 这个链接克隆的就不能用了 还有一个是完整克隆与之前的母本虚拟机完全独立 即使母本被删了 这个完整克隆出来的也还能用 完整克隆的缺点就是占用空间大 链接克隆占用空间小

工具栏第四个虚拟机 设置 我要单独来说明一下 如果你新建好了虚拟机 这个设置可以在两个地方打开看图片操作

点开设置 里面可以对硬件进行 一一编辑 可以编辑 内存 cpu 硬盘 等等。

在这里重点说一下 网络适配器 网络适配器可以更改虚拟机的网络连接方式 这里如果你不太会配置 点击NET模式 然后让虚拟机自动联网获取ip就可以了
NAT模式 利用虚拟的NAT设备以及虚拟DHCP服务器来使虚拟机连接外网
桥接模式就是将主机网卡与虚拟机虚拟的网卡利用虚拟网桥进行通信。在桥接的作用下，类似于把物理主机虚拟为一个交换机，所有桥接设置的虚拟机连接到这个交换机的一个接口上，物理主机也同样插在这个交换机当中，所以所有桥接下的网卡与网卡都是交换模式的，相互可以访问而不干扰。在桥接模式下，虚拟机ip地址需要与主机在同一个网段，如果需要联网，则网关与DNS需要与主机网卡一致。
仅主机模式 把虚拟机和真机连在同一局域网下 如果想要联网就需要真机给虚拟机共享网卡

# include <string.h>
void main()
{
char username[15],pwd[15];
printf(“\n 请输入用户名： “ );
gets(username);
printf(“\n 请输入密码： “);
gets(pwd);
if((strcmp(username,”John”)==0)&&(strcmp(pwd,”123456”)==0))
printf(“\n 您已成功登录 \n “);
else
printf(“\n 用户名或密码有误 \n “);
}
